# Protocol Analyze TS 🚀

Проект по анализу и сравнению сетевых протоколов передачи данных на языке TypeScript. Система спроектирована как модульный фреймворк для тестирования различных транспортных протоколов и методов сериализации данных.

## 📌 Обзор проекта

Данный репозиторий содержит реализацию сетевой инфраструктуры "Клиент-Сервер", предназначенную для проведения замеров производительности (latency, throughput) и анализа структуры пакетов при передаче данных по сети.

Главная особенность — **полная абстракция от конкретного протокола**. Благодаря использованию интерфейсов, логика приложения не зависит от того, передаются ли данные по TCP или UDP.

## 🏗 Архитектура системы

Проект построен на принципах модульности и инверсии зависимостей:

### 1. Транспортный уровень (Transport Layer)
Все протоколы реализуют единый интерфейс `transportClient`, что позволяет "горячую" замену транспортного уровня в коде:
*   **TCP Protocol**: Реализация потоковой передачи данных с гарантией доставки на базе модуля `node:net`.
*   **UDP Protocol**: Реализация на базе датаграмм через модуль `node:dgram`, ориентированная на максимальную скорость без подтверждения доставки.
*   **Архитектурный задел**: Структура проекта позволяет интегрировать современные протоколы (например, QUIC или SCTP) без изменения бизнес-логики.

### 2. Слой сериализации (Serialization)
Для гибкой работы с данными используется контракт `serializerClient`:
*   Метод `serialize`: Преобразование объектов в бинарный вид (`Buffer`).
*   Метод `deserialize`: Обратное преобразование байт-кода в исходные структуры данных.
*   Поддержка кастомных форматов данных (определение типа сообщения, длины полей и полезной нагрузки).

## 🛠 Технологии

*   **Runtime**: Node.js
*   **Language**: TypeScript
*   **Network Stack**: Native Node.js `net` & `dgram` modules

## 📂 Структура репозитория

*   `src/protocols/` — реализации сетевых протоколов.
*   `src/serializers/` — логика упаковки и распаковки данных по заданным контрактам.
*   `src/` — реализация клинта + сервера.
